# システム仕様書 (Knowledge Base System)

## 1. システム概要
本システムは、製造・保守現場における不具合、点検、修理等のナレッジを蓄積・共有するためのWebアプリケーションです。AIチャットによる自然言語検索と、詳細なマスタ管理機能を備えています。

## 2. 技術スタック
- **フロントエンド**: React (TypeScript), Vite
- **デザイン**: Liquid Glass UI (リキッドガラス、ダークモード、黒を基調としたプレミアムデザイン)
- **バックエンド/DB**: Supabase (PostgreSQL, Auth, Storage)
- **インフラ**: Vercel (デプロイ環境)
- **認証**: Azure AD / Microsoft SSO および メール・パスワード認証

## 3. 主要機能
### 3.1 ナレッジ検索・閲覧
- **サイドバー検索**: キーワードによる即時フィルタリング。
- **タグ・カテゴリ検索**: 登録済みのマスタカテゴリやタグによる絞り込み。
- **AIアシスタント**: 画面右下のチャットボットによる、ナレッジベース内の自然言語検索と回答。

### 3.2 ナレッジ登録・編集
- **エディタ**: タイトル、内容（Markdown形式）、号機、物件名、カテゴリ、現象、タグの編集。
- **保存機能**: 非同期更新および履歴管理。

### 3.3 管理者機能 (マスタ管理)
- **現象マスタ**: ナレッジに紐付ける「現象」のリスト管理。
- **カテゴリマスタ**: ナレッジの分類リスト管理。
- **ユーザーロール管理**: ユーザーごとの権限レベル（Viewer, User, Manager, Master）の制御。
- **ユーザー追加**: 管理者による新規ユーザーのメールアドレスおよびロールの事前登録。

## 4. データ構造 (Supabase主要テーブル)
### 4.1 `knowledge` テーブル
ナレッジの本体データを格納します。

| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| id | text | プライマリキー |
| title | text | ナレッジのタイトル |
| content | text | 詳細内容 |
| category | text | カテゴリ名 |
| incidents | jsonb | phenomenon (現象) のリスト |
| tags | jsonb | タグリスト |
| status | text | 解決状況 (solved / unsolved) |
| author | text | 作成者名 |
| updated_at | timestamptz | 最終更新日時 |

### 4.2 `profiles` テーブル
ユーザーのプロフィールと権限を管理します。

| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| id | uuid | プライマリキー (Auth.users.id と紐付け) |
| email | text | メールアドレス |
| display_name | text | 表示名 |
| knl_role | text | 権限 (viewer, user, manager, master) |

## 5. 権限定義
- **Viewer (閲覧者)**: ブラウズ、検索、閲覧のみ可能。
- **User (編集者)**: ナレッジの新規登録・編集が可能。
- **Manager (管理者)**: 全てのナレッジの編集、マスタデータの管理が可能。
- **Master (最上位者)**: システム全域の設定および権限制御が可能。
