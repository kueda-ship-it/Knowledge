# Project Memory: Knowledge Management System

## 概要
*   **目的**: 現場のナレッジを効率的に収集・共有・評価し、品質の高い情報資産を構築する。
*   **技術スタック**:
    *   **Frontend**: React (Vite), TypeScript
    *   **Design**: Glassmorphism (透明感のあるデザイン), Vanilla CSS
    *   **Backend**: Supabase (PostgreSQL, Realtime, Profiles)
    *   **Storage**: Microsoft OneDrive (SSO統合によるファイル管理)
    *   **UI Components**: Lucide-React Icons

## 現在のステータス
*   [x] ログイン機能 (Profilesテーブル連携)
*   [x] ナレッジ作成・一覧・編集 (カテゴリー・タグ付け)
*   [x] OneDrive ファイル添付・プレビュー機能
*   [x] **[NEW]** フィードバック・リアクション機能 (いいね・違うよ)
*   [x] **[NEW]** 差分履歴 (Diff) 表示機能
*   [x] **[NEW]** 通知システム (通知ベル)
*   [x] **[NEW]** 評価確認ダッシュボード (ランク付け)

## 重要な決定事項
*   **差分管理**: 最終成果物だけでなく、編集過程（誰がどこを修正したか）を可視化することで、情報の信頼性を担保する。
*   **指摘コメント**: 「違うよ！」リアクション時にはコメントを必須または入力推奨とし、ハルシネーションの防止とナレッジの洗練を促す。
*   **通知ロジック**: 投稿者に対し、リアクションや編集があった際に即座にベルアイコンで通知する。
*   **UI統一**: ホバーエフェクトは「浮遊感（transform: translateY(-2px)）」と「グロー効果（box-shadow）」を組み合わせたプレミアムな質感を維持する。

## UI/UXのこだわり
*   **透明感**: Glassmorphismを採用。背景との重なりを意識した視覚的な奥行きを重視。
*   **フィードバックの視認性**:
    *   「いいね！」 = 成功/承認の色 ( var(--primary) / Blue )
    *   「違うよ！」 = 警告/要修正の色 ( #ef4444 / Red )
*   **操作感**: 戻るボタンや閉じるボタンも「押している感覚」をホバー時の輝きで表現。

## アンチパターン/教訓
*   **直接的なDBスキーマ変更**: MCP経由のSQL実行が権限エラーになる場合があるため、DDL操作はSupabaseダッシュボードでの手動実行を推奨する手順を残す。
*   **アイコンの一貫性**: lucide-react以外のアイコンを混ぜず、一貫したラインウェイトを守る。

## 既知の課題/技術的負債
*   **通知の永続化**: 現在はポーリングベース。よりリアルタイム性を高めるには Supabase Realtime の利用を検討。
*   **検索性能**: ナレッジ数が増大した場合、全文検索（pgroonga等）の導入が必要。
*   **SQL実行**: SQL作成済みのテーブルが確実に存在することを前提としているため、初回の環境構築手順をドキュメント化し続ける必要がある。
